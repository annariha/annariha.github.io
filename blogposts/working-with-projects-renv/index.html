Installing renv

Starting from a fresh installation of R and Rstudio, the first step is to install the renv package and then following the recommendations given in the package’s documentation.

install.packages("renv")

Creating a project and connecting to Github repository

renv::init()

renv::init() checks all scripts and files containing R code for dependencies and automatically installs most of the packages to be recorded in the lockfile. According to the output provided by the first run of renv::init(), the yaml package is required to parse dependencies within Quarto Markdown files and needs to be installed, for example, with renv::install("yaml", library = "renv/library/R-X.X/your-os") or install.packages("yaml", lib = "renv/library/R-X.X/your-os"), where “X.X” is a placeholder for your R version and “your-os” could be, for example, “x86_64-pc-linux-gnu” if you are using a Linux-based operating system like Ubuntu.

# initialise renv: creates renv folder, checks dependencies etc. 
renv::init()
# to check dependencies in .qmd-files
renv::install("yaml", library = "renv/library/R-4.3/x86_64-pc-linux-gnu")
# run again to see whether there are additional dependencies in any of the .qmd-files
renv::init()

Manually installing packages into the project library

An example of some of the output provided after running renv::init() for the first time:

The following required packages are not installed:- MASS [required by ggplot2]- mgcv [required by brms, ggplot2]Consider reinstalling these packages before snapshotting the lockfile.

Note that if you simply call install.packages(), the packages are installed in your general R library. What you (probably) want is to install them in your project library and add them to your lockfile there as illustrated above for the yaml package. When you are in your project folder, your project library can be found in a directory similar to “renv/library/R-X.X/your-os” where “X.X” is a placeholder for your R version and “your-os” could be, for example, “x86_64-pc-linux-gnu” if you are using a Linux-based operating system like Ubuntu. There are several ways to install packages in your project library.

In most of my projects, some of the widely used packages like Rcpp, MASS, mgcv, nlme, codetools or boot were not installed automatically by renv::init() and I installed them manually using:

renv::install(c("codetools", "MASS", "mgcv", "nlme", "Rcpp"), library = "renv/library/R-4.3/x86_64-pc-linux-gnu")

For example, for installing cmdstanr in your project library using install.packages() as recommended here, replace “renv/library/R-4.3/x86_64-pc-linux-gnu” in the code below with your path and run something similar to

install.packages("cmdstanr", 
                 repos = c("https://mc-stan.org/r-packages/", getOption("repos")), 
                 lib = "renv/library/R-4.3/x86_64-pc-linux-gnu")

With renv::install it is also possible to provide user name and repository to install packages from Github. Here is an example installing the priorsense package:

renv::install("n-kall/priorsense", library = "renv/library/R-4.3/x86_64-pc-linux-gnu")

After that, one can call renv::init() again and select the option to re-initialise the project with a new library. If all dependencies are resolved, this will write the lockfile to renv.lock in the project directory. If the lockfile already exists, one can instead also run renv::snapshot() instead to add newly installed packages to the lockfile.

Collaborating

To collaborate with others, one needs to commit renv.lock, .Rprofile, renv/settings.json and renv/activate.R which then allows others to recreate the project environment. Every time the lockfile is updated, you need to commit this change and notify your collaborators to run renv::restore() next time they work on the project.
